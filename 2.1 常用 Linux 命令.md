### 2.1 常用 Linux 命令

作为一名测试人员，Linux 命令是日常工作中不可或缺的工具，尤其在搭建测试环境、日志分析、自动化测试和环境调试等场景中。

**一、文件与目录操作**

- **`ls`**（list）：列出当前目录下的文件和文件夹。常用选项 `-l` 显示详细列表（权限、大小、时间等），`-a` 显示隐藏文件。例如 `ls -la` 可查看测试环境目录下所有内容，确认文件是否存在。

- **`cd`**（change directory）：切换工作目录，进入指定路径。例如 `cd /var/log` 可切换到日志目录查看服务器日志。

- **`pwd`**（print working directory）：显示当前工作目录的绝对路径，例如输出 `/usr/local/test`，方便确认当前位置再进行文件操作。

- **`cp`**（copy）：复制文件或目录，复制目录需加 `-r` 选项（递归复制）。例如 `cp test.sh /tmp/` 可将脚本备份到 `/tmp` 目录。

- **`mv`**（move）：移动文件或重命名文件。例如 `mv oldname.sh newname.sh` 可重命名测试脚本。

- **`rm`**（remove）：删除文件或目录，常用 `-r`（递归删除目录）和 `-f`（强制删除）选项。用于清理临时文件或测试环境。

- **`touch`**：创建空文件，或更新已有文件的时间戳。例如 `touch test.txt` 可快速创建测试用临时文件。

**二、权限管理**

- **`chmod`**（change mode）：修改文件权限。例如 `chmod 755 test.sh` 可给脚本设置“所有者可读写执行、组和其他用户可读执行”的权限，常用于给测试脚本添加可执行权限。

- **`chown`**（change owner）：修改文件的所有者。例如 `chown user test.sh` 可将文件所有者改为指定用户，用于调整测试文件的归属。

**三、远程连接与文件传输**

- **`ssh`**（secure shell）：安全远程连接服务器，需要用户名和 IP 地址。例如 `ssh user@192.168.1.1` 可远程登录测试服务器进行操作。

- **`scp`**（secure copy）：在本地和服务器之间安全传输文件。例如 `scp test.sh user@192.168.1.1:/tmp/` 可将本地的测试脚本上传到服务器的 `/tmp` 目录。

**四、压缩与解压**

- **`tar`**（tape archive）：打包或解压文件，常用选项组合：`-z`（gzip 压缩）、`-x`（解压）、`-v`（显示过程）、`-f`（指定文件）。例如 `tar -zxvf archive.tar.gz` 可解压 `.tar.gz` 格式的测试环境依赖包。

- **`unzip`**：专门解压 `.zip` 格式的压缩文件。例如 `unzip archive.zip` 用于解压测试用的 zip 包。

**五、网络请求**

- **`curl`**：发送 HTTP 请求，用于测试接口或下载文件。例如 `curl http://api.example.com` 可快速验证接口是否正常返回。

- **`wget`**（web get）：从网络下载文件。例如 `wget http://example.com/file.zip` 可用于下载测试依赖包。

**六、系统监控与资源查看**

- **`top`**：实时查看系统资源使用情况（CPU、内存、进程等）。直接运行 `top` 可找到高负载进程，用于监控服务器性能，排查测试瓶颈。

- **`htop`**：`top` 的升级版，界面更友好，支持颜色和交互操作。运行 `htop` 后可按 `F6` 排序，快速定位资源占用高的进程，适合性能测试时监控服务器。

- **`df`**（disk free）：查看磁盘分区使用情况，`-h` 选项以人类可读格式显示。例如 `df -h` 可检查 `/var/log` 分区空间是否充足，避免日志写满。

- **`du`**（disk usage）：查看指定目录或文件的磁盘占用大小，常用 `-h` 选项。用于排查哪个日志文件夹占用空间过大。

**七、进程管理**

- **`ps`**（process status）：查看当前进程快照。常用 `ps -ef` 显示所有进程，结合 `grep` 过滤，例如 `ps -ef | grep java` 可检查测试服务器上的 Java 进程是否运行。
- **`kill`**：终止指定进程，`-9` 选项为强制终止。例如 `kill -9 12345` 可强制结束 PID 为 12345 的异常测试进程。

**八、目录创建与文件查看**

- **`mkdir`**（make directory）：创建新目录。例如 `mkdir test_env` 可在当前路径下创建 `test_env` 目录，用于存放测试文件，适合搭建测试环境时使用。

- **`cat`**（concatenate）：查看文件内容，或将多个文件合并输出。例如 `cat error.log` 可直接在终端显示 `error.log` 的完整内容，适合查看较小的测试日志或配置文件。

- **`more`**：分页显示文件内容，按空格键向下翻页。例如 `more access.log` 可分页查看较大的日志文件，避免一次性加载过多内容导致卡顿。

- **`less`**：比 `more` 更灵活的分页查看工具，支持上下方向键翻页，还支持 `/关键字` 进行搜索。例如 `less server.log` 打开日志后，输入 `/error` 可快速定位错误信息，是分析大日志文件最常用的命令。

- **`grep`**（global regular expression print）：在文件中搜索匹配关键字的行，支持正则表达式。常用选项 `-i` 忽略大小写，`-n` 显示行号，`-r` 递归搜索目录。例如 `grep -in "error" server.log` 可在 `server.log` 中查找所有包含 "error"（不区分大小写）的行并显示行号，是定位日志错误的核心命令。

- **`head`**：查看文件开头部分，默认显示前 10 行，可用 `-n` 指定行数。例如 `head -n 5 access.log` 可查看日志的前 5 行，适合检查日志格式或时间戳起始位置。

- **`tail`**：查看文件末尾部分，默认显示最后 10 行。常用 `-f` 实时跟踪文件新增内容（非常适合监控实时日志），`-n` 指定行数。例如 `tail -f error.log` 可实时滚动显示 `error.log` 的最新日志，上线后监控异常输出非常有用。

**九、文件查找与命令定位**

- **`find`**：按条件在指定目录下查找文件，支持按名称、大小、修改时间等过滤。例如 `find /var/log -name "*.log"` 可找出 `/var/log` 目录下所有以 `.log` 结尾的文件，常用于查找测试日志或配置文件的位置。

- **`which`**：查找可执行命令的绝对路径。例如 `which python3` 输出 `/usr/bin/python3`，用于确认测试工具（如 Python、Java）的安装路径，方便在脚本中引用。
